# Radio Sky Map

Frontend-only single page application for visualizing bright radio and astronomical sources on a circular sky plot.

The app is built with React, TypeScript, and Vite. It runs entirely in the browser, has no backend, and builds to static assets suitable for GitHub Pages, Netlify, or Vercel.

## Features

- Circular SVG sky plot with rings, axes, source markers, labels, and clipping
- `Sin` and `Azimuthal` projection modes
- Site, date, time, and timezone controls
- Built-in source catalog stored locally in code
- Dynamic `Sun`, `Moon`, and `Jupiter` positions calculated client-side
- Dotted Sun track
- Blurred Milky Way path
- Desktop-first layout constrained to the viewport height

## Tech Stack

- React 18
- TypeScript
- Vite
- SVG rendering
- [`astronomy-engine`](https://www.npmjs.com/package/astronomy-engine) for topocentric Sun/Moon/Jupiter ephemerides

## Getting Started

```bash
npm install
npm run dev
```

The default Vite dev server runs on `http://127.0.0.1:5173/` unless that port is already in use.

## Build

```bash
npm run build
```

Production assets are emitted to `dist/`.

## Deployment

The Vite config uses a relative asset base so the built site can be served from a subpath.

- GitHub Pages: deploy the contents of `dist/`
- Netlify: publish `dist/`
- Vercel: use the static build output in `dist/`

- Herc A
- Sgr A
- Sun

Catalog data lives in `src/data/sources.ts` and is easy to extend.

## Coordinate and Projection Notes

- Longitude is east-positive and west-negative throughout the app.
- The observing site is selected client-side from local metadata.
- Fixed sources use stored RA/Dec values.
- Dynamic bodies use topocentric equatorial coordinates from `astronomy-engine`.
- Horizontal coordinates are derived from local sidereal time and converted to altitude/azimuth.
- The rendered plot uses a mirrored sky-view convention to match telescope visibility style displays.

## Milky Way and Tracks

- The Sun track is generated by sampling around the selected time and projecting the samples into the active view.
- The Milky Way path is generated from samples along galactic latitude `b = 0`, converted to equatorial coordinates, then projected into the same plot.

## Notes

- This is a static SPA. There is no server, no database, no auth, and no required external API.
- The astronomy math is modular so projection or ephemeris accuracy can be improved later without changing the UI structure.
